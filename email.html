Email Microservice Architecture Diagram

Client App
- Sends Email JSON via RabbitMQ
- UI to Compose/View Emails
↓
RabbitMQ Queue
- Receives email payload from Client
- Format: JSON, HTML, Text, EML
- With/Without Attachments
↓
Email Microservice (Laravel + PHP)
Queue Listener
SMTP Sender (SendGrid, SES, Custom SMTP)
IMAP/POP3 Poller (for A, B, C)
Email Parser (MIME, EML)
Encoding Handler (Base64, quoted-printable)
Attachment Extractor
Inline CID Resolver
Error Handler + Retry Logic
↓
Local Database
- Email metadata (to, from, subject, etc.)
- User folders (inbox, sent, spam)
- Indexed for fast search
File/Blob Storage
- Raw EML / MIME files
- Attachments
- Embedded Images
↑
User A
Pulls from IMAP
View Inbox / Send
User B
Pulls from IMAP
View Inbox / Send
User C
Pulls from IMAP
View Inbox / Send

Summary: The client places a request on RabbitMQ. Email Microservice consumes it, sends email via SMTP, and polls user inboxes via IMAP/POP3 to update local DB. All MIME formats and encodings are supported, with Laravel PHP stack powering the microservice logic.
